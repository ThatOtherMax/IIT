Version Control Systems Overview

What is Version Control?

Version Control Systems (VCS) are tools that help software teams manage changes to source code over time.

Types of Version Control Systems

  • Local VCS - Simple database that keeps all changes to files under revision control
  • Centralized VCS - Single server contains all versioned files; clients check out files from central place
  • Distributed VCS - Clients fully mirror the repository, including its full history

Why Use Version Control?

  • Track changes and history
  • Collaborate with multiple developers
  • Recover previous versions
  • Branch and merge code safely
